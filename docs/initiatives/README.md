# Iniciativas del Proyecto YouTube

La línea base de producto es [MVP_BASELINE.md](../product/MVP_BASELINE.md) y el Plan 001 es el plan de entrega vigente: el MVP define producto mínimo; el plan define cómo y cuándo se entrega.

La captura es progresiva. `recommendation` no es `decision`, y clasificación no es autorización. Cada iniciativa canónica vive en `records/`; `REGISTRY.yaml` solo indexa; `decisions/` conserva decisiones explícitas. La autoridad es la definida en `docs/ALCANCE_Y_COORDINACION_EQUIPOS.md`.

En revisión de baseline, planificación o alcance se comprueban MVP, plan, duplicados y puntos de extensión. Ninguna iniciativa se implementa por estar registrada: `implementation_authorized: false` hasta autorización completa. Regla para agentes: localizar MVP y plan, capturar proporcionalmente, no ampliar la misión y continuar el alcance autorizado.
