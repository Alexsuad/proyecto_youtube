# Reglas NotebookLM (cuando se use)
Versión: 1.0
Fecha: 19/02/2026

---

## 1) NotebookLM es “cerebro”, no “guionista”
NotebookLM se usa para:
- investigación con fuentes
- guardar conocimiento acumulado
- generar síntesis consultables

NotebookLM no debe reemplazar:
- estructura del guion
- reglas editoriales
- QA de estilo

---

## 2) Cuándo se usa NotebookLM
Se recomienda usarlo en:
- Fase 1 (Research): para investigar tema y obras con fuentes. **IMPORTANTE:** Usar cuadernos TEMPORALES (Carril de Buffer) que se descartan o ignoran tras el cierre.
- Biblioteca acumulativa (Cerebro Central): guardar únicamente piezas finales (Guiones, Shorts, Pack) para consulta futura. No subir investigación cruda aquí.

---

## 3) Qué debe quedar trazado en outputs
Cuando se use NotebookLM, en output/01_research_bruto.md se debe indicar:
- qué tipo de fuentes se consultaron (sin inventar)
- qué conclusiones salieron
- qué quedó pendiente

---

## 4) Si NotebookLM no está disponible
Si falla o no está conectado:
- seguir el pipeline con Antigravity
- marcar límites de la investigación
- dejar PENDIENTES claros
